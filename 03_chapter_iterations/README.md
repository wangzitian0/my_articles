# 03_章节迭代阶段

## 阶段目标
- 与用户逐章对话，确认每章的逻辑、案例、数据与语气。
- 输出可直接粘贴到正文的章节草稿，并附节级质检结果。

## 交付物
1. `articles/<topic>/chapters/chapter_<n>.md`：对应章节的最新版本。
2. `articles/<topic>/chapters/chapter_<n>_qa.md`：该章节的质检记录（章级 + 节级）。

## 使用说明
1. 使用 `chapter_prompt.md` 引导对话：每次只处理一个章节，明确痛点和调整项。
2. 每节写完即引用章/节质检清单，记录是否通过、待改项。
3. 迭代结束后，将章级摘要同步到 `insights.md` 以便全局一致。

## 相关文件
- `chapter_prompt.md`：逐章迭代提示词模板。
- `../05_quality_prompts/quality_checklists.md`：章级与节级质检列表。

## 进入下一阶段前的检查
- [ ] 当前章全部节都有通过的节级质检记录。
- [ ] 视觉元素与互动引导已落位或注明“待插图”。
- [ ] 与用户确认“我”字计数仍在 15 次以内。
