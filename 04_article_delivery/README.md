# 04_成稿交付阶段

## 阶段目标
- 将已确认的章节草稿合并为 3,500-4,000 字的公众号文章。
- 确认视觉元素、互动引导、行动清单全部到位。

## 交付物
1. `articles/<topic>/article.md`：最终全文草稿。
2. `articles/<topic>/visual_plan.md`：图表/时间轴/书单/流程图/行动清单的具体素材或备注。

## 使用说明
1. 先参考 `article_template_v5.md` 确认整体结构与字数分配。
2. 使用 `article_prompt.md` 生成首稿，输入包括章节草稿、案例矩阵、质检记录。
3. 生成后立即运行 `../05_quality_prompts/quality_checklists.md` 中的全文检查项。
4. 将视觉素材与互动话术同步到 `visual_plan.md`，便于设计/运营衔接。

## 相关文件
- `article_template_v5.md`：写作与排版的详细规范。
- `article_prompt.md`：用于生成整篇文章的提示词。

## 进入下一阶段前的检查
- [ ] 全文 5 章顺序完整，每章 2-4 节。
- [ ] “我”字使用 ≤ 15 次，若接近上限需在文末提醒。
- [ ] 行动清单含指标、触发条件、操作、检查节奏。
