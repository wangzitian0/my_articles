# 提示词更新指南

1. **收集问题**  
   - 来源：质检反馈、用户对话、写作中遇到的重复性困难。  
   - 记录：在 `prompt_update_log.md` 中描述情境、影响与优先级。

2. **定位阶段**  
   - 判断问题属于哪个工作流阶段（01-05）。  
   - 若跨阶段，拆分为多个子问题分别处理。

3. **拟定改动**  
   - 修改相应阶段的提示词或说明文档。  
   - 添加示例、更清晰的输入要求或质检提醒。

4. **验证**  
   - 用新提示词跑一次小样或过去案例，确认问题得到缓解。  
   - 在 `prompt_update_log.md` 写明验证结论。

5. **同步**  
   - 更新 `agent.md` 中的工作流说明，确保 Agent 行为与最新提示词一致。  
   - 在 README 中补充新能力或注意事项（如有必要）。
