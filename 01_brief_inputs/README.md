# 01_输入提纲阶段

## 阶段目标
- 收集用户提供的提纲、案例、核心观点，并核对 3 秒钩子与 3 重痛点。
- 明确缺失素材（报告/案例/书籍）与所需数据四要素，为后续归纳做准备。

## 交付物
1. `articles/<topic>/outline.md`：整理后的章节要点列表，记录当前素材覆盖度。
2. `articles/<topic>/gap_list.md`：尚需补充的数据、案例、来源。

## 使用说明
1. 与用户确认输入后，使用 `outline_prompt.md` 生成结构化提纲。
2. 在 `outline.md` 中标记每节对应的视觉元素和互动引导占位。
3. 若某个数据源不完整，立即放入 `gap_list.md` 并说明缺失项。

## 相关文件
- `outline_prompt.md`：用于快速生成符合 V5 模板的提纲，包含视觉元素与互动引导占位要求。

## 进入下一阶段前的检查
- [ ] 5 章骨架已列出，且每章 2-3 节。
- [ ] 每节都写明所需案例主体与数据占位。
- [ ] 缺失素材已集中记录，方便阶段二补齐。
