# 02_素材归纳阶段

## 阶段目标
- 将输入的提纲、案例、核心观点进行归纳，输出可执行的章节陈述。
- 补齐案例五要素与数据四要素，并将缺口反馈给用户。

## 交付物
1. `articles/<topic>/cases.md`：以表格列出每个案例的主体、时间、动作、数据、结果、来源。
2. `articles/<topic>/insights.md`：归纳核心矛盾、洞察与可验证数据链接。

## 使用说明
1. 先用 `case_prompt.md` 收集/补写详细案例。
2. 再通过 `synthesis_prompt.md` 将提纲与案例融合成章节洞察，标注需要重点讨论的部分。
3. 归纳过程中保持“我”字使用计数，为后续写作预留额度.

## 相关文件
- `case_prompt.md`：标准化案例采集模板。
- `synthesis_prompt.md`：将提纲+案例+观点汇整为章节摘要的提示词。

## 进入下一阶段前的检查
- [ ] 每个案例具备主体、时间、动作、数据、结果。
- [ ] 数据均附四要素（数值/时间/主体/来源）。
- [ ] `insights.md` 中写清每章核心矛盾与待确认事项.
